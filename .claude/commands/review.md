# コードレビュー依頼

## 対象
$ARGUMENTS

## 対象が指定されていない場合
引数が空の場合は、以下のコマンドで変更ファイルを取得してレビュー対象としてください:
!`git diff --name-only HEAD 2>/dev/null || echo ""`

## レビュー実行

`code-reviewer` サブエージェントを使用して、以下の4観点でコードをレビューしてください:

1. **Next.js ベストプラクティス**
   - Server/Client Components の適切な使い分け
   - データフェッチの方法
   - Image 最適化
   - メタデータ設定

2. **React ベストプラクティス**
   - Hooks のルール遵守
   - useEffect の依存配列
   - key の適切な使用
   - イミュータビリティ
   - 型定義

3. **CSS ベストプラクティス**
   - インラインスタイルの回避
   - マジックナンバーの回避
   - セマンティックな命名
   - レスポンシブデザイン

4. **アクセシビリティ**
   - セマンティック HTML
   - キーボード操作対応
   - 画像の alt 属性
   - フォームのラベル
   - コントラスト比
   - フォーカス管理

## 出力形式

問題点を重大度（Critical / Warning / Info）別に分類し、以下の情報を含めて報告してください:
- ファイル名と行番号
- 問題の説明
- 具体的な改善案（コード例付き）
