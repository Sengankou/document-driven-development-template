# Purpose

マルチモーダルな大規模言語モデルを利用し、会議支援をリアルタイムに行うwebアプリケーションを構築する。
フロントエンドにおいて、画面共有を行い、その画面の音声と定期的な画像を取得しバックエンドへ送信する。
バックエンドにおいて以下を実行することで、リアルタイムな会議コンテキストの作成を行い、常に会議の画面を含めた内容を理解して、適切にユーザの質問に回答・アドバイスできるチャット体験を実現する。
1. 音声文字起こしを行ってテキストデータ化
2. 重要画面画像の選別とVLMによる解釈を行って、特に重要かつテキストで説明しづらい情報量のある画面のみを永続化し、いつでも呼び出せるようにする。
3. ユーザが対話するAIには常に画面・音声を含めた会議全体のコンテキストを注入し、対話AIはリアルタイムに会議の内容に基づいた適切な回答・アドバイスを行うことができる。

【注意事項：2025/12/19時点】
- 現在は、画面情報を常に送り続けているGemini Live APIに定期要約させることで画面情報を含めたテキスト要約生成＝コンテキスト作成を行わせているが、今後はGemini Live APIを廃止し、画面画像の選別・VLMを用いた解釈・永続化などの画面情報に関するコンテキストの作成はバックエンド側のロジックにおいて実装していく方針である。
