# 90_archive - アーカイブ

完了したフェーズの記録を保存するディレクトリです。

## 特性
- **アーカイブ**: 完了したフェーズの振り返り・変更記録
- **対象読者**: 将来のプロジェクトメンバー、振り返り時

## ディレクトリ構造

```
90_archive/
├── README.md           # このファイル
├── phase1/
│   ├── retrospective.md   # 振り返り
│   └── changelog.md       # 変更サマリ
├── phase2/
│   └── ...
└── ...
```

## フェーズ完了時に作成するドキュメント

### retrospective.md（振り返り）

```markdown
# Phase 1 振り返り

## 期間
YYYY-MM-DD 〜 YYYY-MM-DD

## 達成したこと
- （完了した機能・目標）

## 達成できなかったこと
- （未完了の項目と理由）

## 学んだこと
- （技術的な学び）
- （プロセスの学び）

## 次フェーズへの改善点
- （改善すべき点）

## メトリクス
- 実装した機能数:
- 発見されたバグ数:
- 解決されたバグ数:
- ...
```

### changelog.md（変更サマリ）

```markdown
# Phase 1 変更サマリ

## 追加された機能
- [US-001] ユーザー登録機能
- [US-002] ログイン機能
- ...

## 変更されたアーキテクチャ決定
- [ADR-003] 認証方式をJWTに変更

## 追加されたAPI
- [API-001] POST /auth/register
- [API-002] POST /auth/login
- ...

## 追加された画面
- [SCR-001] ログイン画面
- [SCR-002] 登録画面
- ...

## 既知の問題・技術的負債
- （残った課題）
```

## アーカイブの目的

1. **知識の保存**: 将来のメンバーが経緯を理解できる
2. **振り返り**: 継続的改善のための材料
3. **監査対応**: プロジェクト履歴の記録
