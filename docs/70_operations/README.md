# 70_operations - 運用

システムの運用・保守に関するドキュメント群です。

## 特性
- **累積的**: 運用方針は全フェーズで一貫して維持
- **対象読者**: 運用チーム、SRE、開発チーム

## ドキュメント一覧

| ファイル | 目的 | 更新タイミング |
|---------|------|---------------|
| [deployment.md](deployment.md) | デプロイ手順・環境構成・CI/CD | デプロイ方式変更時 |
| [backup_recovery.md](backup_recovery.md) | バックアップ・リカバリ手順 | バックアップ方針変更時 |
| [maintenance.md](maintenance.md) | 運用ポリシー・監視・アラート | 運用方針変更時 |

## 必要に応じて追加

プロジェクトの特性に応じて以下を追加:

- `runbook.md` - 運用手順書（障害対応等）
- `scaling.md` - スケーリング方針
- `disaster_recovery.md` - 災害復旧計画
- `sla.md` - SLA/SLO定義

## 運用ドキュメントの重要性

運用ドキュメントは以下の場面で参照されます:

1. **デプロイ時**: deployment.md
2. **障害発生時**: maintenance.md, backup_recovery.md
3. **定期メンテナンス時**: maintenance.md
4. **監査時**: 全ドキュメント

## チェックリスト

運用開始前に以下を確認:

- [ ] デプロイ手順が文書化されている
- [ ] バックアップが自動化されている
- [ ] リカバリ手順がテストされている
- [ ] 監視・アラートが設定されている
- [ ] オンコール体制が整備されている
